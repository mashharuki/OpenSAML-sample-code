# Project Rules for Antigravity

このファイルは、本プロジェクトにおける開発ルールとAIアシスタント(Antigravity)向けのガイドラインを定義します。

## 1. プロジェクト概要
OpenSAML 5を使用したSpring Bootサンプルアプリケーションです。

## 2. テクノロジースタック
- **言語**: Java 21 (`java.version` 21)
- **フレームワーク**: Spring Boot 3.3.0
- **主要ライブラリ**: OpenSAML 5.1.6
- **ビルドツール**: Maven

## 3. ディレクトリ構造
- **`/backend`**: バックエンドアプリケーションのルートディレクトリ。
  - `pom.xml`: Mavenの設定ファイル。
  - `src/`: ソースコード。
  - **重要**: Java/Maven関連の作業はすべてこのディレクトリ配下で行う必要があります。
- **`/.github/workflows`**: GitHub Actionsの定義ファイル。

## 4. 開発・運用ガイドライン

### コマンド実行時の注意
- `mvn` コマンドを実行する際は、必ず `backend` ディレクトリに移動してから実行してください。
- シェルコマンドの提案時も `cd backend` を含めるか、作業ディレクトリを意識してください。

### 開発スタイル
- Spec駆動開発の原則に従って開発を進めてください。
- t-wadaさんのテスト駆動開発の原則に従って開発を進めてください。

### コードスタイル
- 標準的なJavaおよびSpring Bootのコーディング規約に準拠します。
- クラス名、メソッド名は分かりやすい英語を使用し、必要に応じて日本語でJavadocやコメントを記述してください。
- 必ず分かりやすいコメントを日本語で挿入してください。
- 可読性・カスタマイズ性を考慮した実装を行ってください。

### Git / CI
- **CI**: GitHub Actions (`build.yml`) を使用してビルドとテストを自動化しています。
- `.github/workflows/build.yml` の変更時は、`working-directory: backend` の設定が維持されているか確認してください。

## 5. Antigravity への指示
- ユーザーへの応答は**日本語**で行ってください。
- ファイルを作成・修正する際は、既存のディレクトリ構造（特に `backend` フォルダ）を遵守してください。
- 新しい依存関係を追加する場合は、`backend/pom.xml` を対象としてください。
