# Coding Style and Conventions

## Language and Language-Specific Rules
- **Backend**: Java 21, Spring Boot 3.3.0.
- **Infrastructure**: TypeScript, AWS CDK.

## Communication and Documentation
- **Response Language**: Always respond to the user in **Japanese**.
- **Comments/Javadoc**: Use **Japanese** for code comments and Javadoc.
- **Clarity**: Ensure comments are easy to understand.

## Principles
- **Spec-driven development**: Follow the principles of Spec-driven development.
- **TDD**: Follow Test-Driven Development principles (t-wada style).
- **Working Directory**: Always be mindful of the working directory. Use `backend/` for Maven commands and `cdk/` for CDK commands.

## Code Style
- Follow standard Java and Spring Boot coding conventions.
- Use descriptive English for class and method names.
